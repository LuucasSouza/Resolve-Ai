# v0.1.0-alpha Phase 18 GO/NO-GO Decision

## Decision

**RELEASE CANDIDATE READY (90/100) — aguardando aprovação explícita do mantenedor.**

Não é GO. Não é stable. Nada foi publicado.

## Why Not GO (stable ou publicação imediata)

- Não houve validação humana real adicional nesta fase: Non-Technical Builder segue pendente e Professional Engineer dedicado segue insuficiente.
- A política do projeto (pt272, ADR-0187, ADR-0263) proíbe declarar stable ou validação ampla sem evidência real.
- Publicação npm, tag Git e GitHub Release exigem o gate do mantenedor (`v0.1.0-alpha-rc/maintainer-approval-gate.md`), que está integralmente desmarcado.

## Why Not NO-GO

- Build, 87/87 testes automatizados, smoke, `npm pack --dry-run` e `git diff --check` passam.
- Auditoria do pacote aprovada: 37 arquivos, 38.5 kB, apenas `dist/`, `README.md` e `package.json`; sem `.resolve-ai/`, `docs/resolve-ai/`, `.env` ou feedbacks privados.
- Toda a documentação de release candidate existe: release notes, known limitations, checklists de npm/tag/distribuição, rollback plan, feedback plan e gate do mantenedor.
- Nenhum safety gate foi removido; `resolver` continua não executando código; sem telemetria.

## Blockers restantes para publicação

1. Aprovação explícita do mantenedor (gate desmarcado).
2. Validação humana real com Non-Technical Builder.
3. Validação dedicada com Professional Engineer.

Os itens 2 e 3 bloqueiam declarar GO/stable; o item 1 bloqueia qualquer execução de publicação, mesmo como alpha.

## Next Action

O mantenedor deve ler `docs/release/v0.1.0-alpha-rc/` e decidir:

- **Aprovar o gate** → Phase 19 — Public Alpha Publication Execution (tag + GitHub pre-release + `npm publish --tag alpha`); ou
- **Priorizar validação humana** → rodar o fluxo Controle Simples de Gastos com um Non-Technical Builder real usando o kit `teste/` antes de publicar; ou
- **Corrigir problemas encontrados** → Phase 18B — Release Candidate Fixes.
