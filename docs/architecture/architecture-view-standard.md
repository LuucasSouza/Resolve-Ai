# Architecture View Standard

## Objetivo

Padronizar views C4-inspired usando Mermaid e documentação repository-native.

## Required Minimum for Non-Trivial Systems

- System Context View.
- Container View.
- Domain Concept View.
- Integration View.
- Deployment/Runtime View.

## Optional

- Component View.
- Sequence diagrams.
- State diagrams.
- Data flow diagrams.
- Threat model diagrams.

## Mermaid Rules

- Diagrams clarify decisions, not decorate documents.
- Prefer multiple smaller diagrams over one mega-diagram.
- Keep diagrams close to explanatory text.
