# Project Zero Detection

## Definition

A project is treated as `novo` when there is no strong local signal of implementation, manifest, source code, README, or framework structure.

Folders generated by Resolve Aí or used only for testing do not count as product structure.

## Ignored Folders

The detector ignores auxiliary folders by default:

- `Resolve-Ai/`
- `resolve-ai/`
- `AI-SEOS/`
- `ai-seos/`
- `node_modules/`
- `.git/`
- `.resolve-ai/`
- `docs/resolve-ai/`
- `teste/`
- `coverage/`
- `dist/`
- `build/`
- `.cache/`

## Recommendation

When the project appears empty or idea-only, `diagnosticar` recommends:

```bash
resolve-ai entrevistar
```

The absence of tests, CI, Git, or source files is not treated as critical by itself in a project-zero state.

## Stack Detection Guardrails

Next.js is detected only with strong evidence:

- `next` dependency;
- `next.config.*`;
- `.next/`;
- npm scripts using `next dev`, `next build`, or `next start`.

Vite React projects are not classified as Next.js just because they have `app/`, `pages/`, React, or TypeScript.
