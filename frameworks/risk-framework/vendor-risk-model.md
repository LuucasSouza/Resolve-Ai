# Vendor Risk Model

## Categories

API lock-in, data lock-in, auth lock-in, runtime lock-in, pricing risk, availability risk, compliance risk, roadmap risk and support risk.

## Exit Strategy Levels

E0 none needed, E1 manual export, E2 export/import scripts, E3 abstraction layer, E4 parallel provider compatibility, E5 human approval and legal/compliance review.
