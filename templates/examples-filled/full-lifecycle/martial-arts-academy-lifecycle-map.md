# Martial Arts Academy Lifecycle Map

## Intake

Problem story and manual process map capture paper, WhatsApp and spreadsheet workflow.

## Discovery

Validate core users, sensitive data, payment-state risk and MVP boundary.

## Product

PRD defines student registry, attendance, manual payment status and dashboard.

## Architecture

Architecture overview selects simple web app, managed database and role-based access.

## Decision

ADRs record stack, data model and auth model.

## Risk

Risk register tracks personal data, payment mistakes and access control.

## Optimization

Defer native app, AI prediction and automated billing.

## Execution

Implement slices: scaffold, students, attendance, payment status, dashboard.

## Documentation

Update README, user guide and operational notes.

## Handoff

Provide implementation and QA handoffs for each slice.

## Reflection

Review whether the dashboard reduced missed follow-ups and payment confusion.
