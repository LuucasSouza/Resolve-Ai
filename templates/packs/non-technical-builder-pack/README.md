# Non-Technical Builder Template Pack

## Objetivo

Ajudar pessoas nao tecnicas a explicar problemas reais sem exigir vocabulario de software.

## Escopo

Cobre problem story, processo manual, stakeholders, solution brief, MVP em linguagem simples, riscos, escopo adiado e handoff em linguagem simples.

## Principios

- Comecar com linguagem do mundo real.
- Traduzir para software apenas depois.
- Nao forcar tecnologia, stack, deploy ou arquitetura no intake.
- Preservar o que deve continuar humano.
