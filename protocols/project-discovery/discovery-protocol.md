# Discovery Protocol

## Objetivo

Definir o procedimento executável para uma sessão ou workflow de discovery.

## Quando usar

Use para novo produto, feature, modernização, arquitetura sem contexto, AI capability ou qualquer iniciativa com usuários, valor ou constraints pouco claros.

## Fases

1. Intake Normalization.
2. Problem Framing.
3. User and Stakeholder Mapping.
4. Current Alternatives Analysis.
5. Business Context.
6. Domain Discovery.
7. Technical Context.
8. Assumption and Constraint Registers.
9. Risk Discovery.
10. MVP Boundary.
11. Validation Plan.
12. Handoff.

## Checklist

- [ ] Raw idea captured.
- [ ] Facts separated from assumptions.
- [ ] Problem statement written.
- [ ] Users and buyer identified.
- [ ] Stakeholders mapped.
- [ ] Alternatives documented.
- [ ] Registers created.
- [ ] MVP boundary defined.
- [ ] Validation and handoff created.
