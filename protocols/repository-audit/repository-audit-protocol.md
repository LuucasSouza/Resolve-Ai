# Repository Audit Protocol

## Purpose

Run a repeatable audit before public-alpha and future releases.

## Input Contract

README, ROADMAP, CHANGELOG, ADR index, docs indexes, registries, release docs and Git status.

## Steps

1. Check navigation.
2. Check structure.
3. Check metadata.
4. Check internal links manually or by tool.
5. Check registries.
6. Check release readiness.
7. Check contributor readiness.
8. Score using 50-point model.
9. Record gaps and blockers.

## Output Contract

Repository audit report, hardening checklist and release recommendation.
