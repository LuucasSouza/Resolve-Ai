# PRD Review Protocol

## Checklist

- [ ] Problem is clear without referencing a specific solution.
- [ ] Users and buyers are separated.
- [ ] Outcomes are measurable.
- [ ] MVP and non-MVP scope are explicit.
- [ ] Requirements are testable.
- [ ] Assumptions are labeled.
- [ ] Constraints are visible.
- [ ] Architecture-relevant details are included.
- [ ] Risks are documented.
- [ ] Open questions are documented.
- [ ] QA can derive test scenarios.
- [ ] Architecture can derive system constraints.
