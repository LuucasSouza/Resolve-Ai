# Architecture Decision Records

## Objetivo

Registrar decisões estruturais do AI-SEOS de forma rastreável.

## Escopo

Inclui decisões sobre arquitetura, documentação, governança, modularidade, agentes, protocolos, licenças e padrões duráveis.

## Princípios

- ADRs aceitas não devem ser reescritas retroativamente.
- Mudanças futuras devem criar nova ADR ou marcar ADR anterior como superseded.
- Trade-offs e plano de reversão devem ser explícitos.

## Responsabilidades

Architecture Maintainers garantem numeração, consistência e referências entre ADRs.
