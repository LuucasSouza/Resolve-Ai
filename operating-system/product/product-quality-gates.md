# Product Quality Gates

| Gate | Requirement |
|---|---|
| P1 Problem Clarity | Problem is user/business framed and not a preselected solution |
| P2 User and Buyer Clarity | Users, buyer, admins, operators and stakeholders are separated |
| P3 Outcome Metrics | Business and user metrics exist with leading/lagging indicators |
| P4 MVP Boundary | MVP, non-MVP and deferred scope are explicit |
| P5 Requirement Quality | Requirements are testable and separated by type |
| P6 Architecture Input Readiness | NFRs, domain concepts, integrations, data and scale signals exist |

## Anti-Patterns

- Feature dump without outcomes.
- MVP as smaller final product.
- Architecture-driven product scope.
- Success defined only as launch.
- Assumptions treated as facts.
