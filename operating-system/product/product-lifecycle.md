# Product Lifecycle

## Fluxo

1. Receive Discovery Outputs.
2. Validate Discovery Completeness.
3. Define Product Problem.
4. Define Users and Jobs.
5. Define Outcomes and Metrics.
6. Map Capabilities.
7. Define MVP Boundary.
8. Define Non-MVP Scope.
9. Write PRD.
10. Create Epic and Feature Map.
11. Define Acceptance Criteria.
12. Prepare Architecture Input Brief.
13. Product Quality Gate.
14. Product Handoff.

## Estados

| State | Meaning | Required Action |
|---|---|---|
| Uninitialized | No discovery output | Request Context Package |
| Discovery Received | Inputs exist | Validate completeness |
| Product Problem Defined | Problem stable enough | Define users, jobs, outcomes |
| MVP Candidate | Initial boundary exists | Challenge scope |
| PRD Draft | PRD exists | Review gates |
| Product Ready | Gates passed | Handoff to Architecture |
| Blocked | Critical input missing | Create Product Input Gap Report |
