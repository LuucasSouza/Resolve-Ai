# AI-SEOS Core Identity

## Objetivo

Definir o que o AI-SEOS sempre deve ser, independentemente do módulo, agente, protocolo, template ou playbook em execução.

## Definição

AI-SEOS é um framework operacional modular, documentation-first, decision-driven e human-governed para engenharia de software com equipes humano + IA.

Ele existe para transformar ideias vagas em iniciativas estruturadas, documentadas, tecnicamente sólidas, seguras, executáveis e evolutivas.

## Escopo

- Linguagem compartilhada de engenharia.
- Lifecycle para transformar ideias em projetos executáveis.
- Protocolos de discovery, arquitetura, decisão, execução, revisão, handoff e reflexão.
- Agentes especializados com responsabilidades e limites explícitos.
- Templates, checklists, ADRs, playbooks e quality gates.
- Governança para evolução open source sustentável.

## Não Escopo

AI-SEOS não é coleção de prompts, chatbot persona, wrapper de geração de código, metodologia rígida, substituto de responsabilidade humana ou museu documental.

## Pilares

- Documentation-first.
- Decision-driven.
- Human-governed.
- Modular by design.
- Secure by design.
- Architecture before implementation.
- Simplicity as default.
- Evolutionary architecture.
- Operational readiness.

## Responsabilidades

Todo módulo deve herdar esta identidade, declarar propósito, limites, entradas, saídas, quality gates, relações com outros módulos, exemplos quando aplicável e Definition of Done.

## Heurísticas

- Clarify before optimizing.
- Prefer reversible decisions early.
- Make assumptions visible.
- Scope is an architectural tool.
- Every shortcut needs a label.
- Handoffs must be lossless.
- Security must be embedded.
- Do not build the future too early.

## Quality Gates

- O artefato distingue fatos, hipóteses, decisões e riscos.
- O artefato pode ser entendido fora da conversa.
- O artefato orienta ação downstream.
- Decisões relevantes apontam para ADRs.

## Próximos passos

- Usar esta identidade como contrato base para todos os agentes e engines.
