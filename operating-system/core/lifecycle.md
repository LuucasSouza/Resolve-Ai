# Lifecycle

## Objetivo

Definir os estágios pelos quais work items passam no AI-SEOS.

## Fluxo

1. Intake.
2. Context Normalization.
3. Discovery.
4. Product Framing.
5. Domain Modeling.
6. Architecture Exploration.
7. Decision Recording.
8. Risk Assessment.
9. Execution Planning.
10. Handoff.
11. Reflection.

## Quality Gates

Um estágio só deve avançar quando suas evidências mínimas existirem. Quando faltarem informações, o sistema deve registrar assumptions, open questions ou bloquear decisões irreversíveis.
