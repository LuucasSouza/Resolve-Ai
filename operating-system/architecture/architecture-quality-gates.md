# Architecture Quality Gates

| Gate | Requirement |
|---|---|
| A1 Input Completeness | PRD, MVP, NFRs, data, integration and security signals available or assumptions documented |
| A2 Drivers Identified | Drivers explicit, constraints separated from preferences, quality attributes prioritized |
| A3 Options Compared | Significant decisions compare at least three options or explain exception |
| A4 Views Documented | Context, container, domain and integration views where applicable |
| A5 Decisions Recorded | ADRs exist for significant choices |
| A6 Implementation Handoff Ready | Boundaries, risks, NFRs and open questions actionable |
